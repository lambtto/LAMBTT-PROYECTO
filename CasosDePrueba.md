# Casos de Prueba – US-01: Registro de gasto individual

> **Autenticación:** Todos los requests requieren el header `Authorization: Bearer <usuario>`.  
> Ejemplo: `Authorization: Bearer juan`

| ID    | Qué se debe hacer (acción / entrada)                                                                                                                                         | Salida esperada                                                                                                      |
|-------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|----------------------------------------------------------------------------------------------------------------------|
| CP-01 | `POST /gastos` con header `Authorization: Bearer juan` y body `{ "monto": 15000, "categoria_id": 1, "fecha": "2026-06-20" }`                                                 | HTTP 201. Body contiene `id`, `monto: 15000`, `categoria_id: 1`, `fecha: "2026-06-20"`.                             |
| CP-02 | `POST /gastos` con header `Authorization: Bearer juan` y body `{ "monto": -500, "categoria_id": 1, "fecha": "2026-06-20" }`                                                  | HTTP 400. Body `{ "error": "El monto es inválido. Debe ser un número mayor a 0." }`.                                |
| CP-03 | `POST /gastos` con header `Authorization: Bearer juan` y body `{ "monto": 5000, "categoria_id": 1, "fecha": "2030-01-01" }`                                                  | HTTP 400. Body `{ "error": "No se permiten fechas posteriores al día actual." }`.                                   |
| CP-04 | `POST /categorias` con header `Authorization: Bearer juan` y body `{ "nombre": "Gimnasio" }`                                                                                 | HTTP 201. Body contiene `id`, `nombre: "Gimnasio"`, `compartida: 0`.                                               |
| CP-05 | `POST /gastos` con header `Authorization: Bearer juan` y body `{ "monto": 8000, "categoria_id": 1, "fecha": "2026-06-20", "descripcion": "Almuerzo de trabajo" }`            | HTTP 201. El campo `descripcion: "Almuerzo de trabajo"` aparece en el body de respuesta.                            |
| CP-06 | `POST /gastos` con header `Authorization: Bearer juan` y body `{ "monto": 3500, "categoria_id": 2, "fecha": "2026-06-20", "metodo_pago": "Tarjeta de crédito" }`             | HTTP 201. El campo `metodo_pago: "Tarjeta de crédito"` aparece en el body de respuesta.                             |
| CP-07 | `GET /gastos` con header `Authorization: Bearer juan`                                                                                                                        | HTTP 200. Array con todos los gastos del usuario `juan`, ordenados por fecha descendente.                           |
| CP-08 | `GET /gastos?categoria_id=1` con header `Authorization: Bearer juan`                                                                                                         | HTTP 200. Array con solo los gastos del usuario `juan` cuyo `categoria_id = 1`.                                     |
| CP-09 | `DELETE /gastos/1` con header `Authorization: Bearer juan` (gasto 1 pertenece a `juan`)                                                                                      | HTTP 200. Body `{ "mensaje": "Gasto eliminado correctamente." }`. El gasto ya no aparece en `GET /gastos`.          |
